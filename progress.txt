## Codebase Patterns
- Use `color_eyre::Result` for error handling throughout
- The app uses async/await with tokio runtime
- CLI uses clap with derive macros for parsing
- Ollama is used for LLM generation with streaming support
- For ratatui Layout constraints, pass arrays directly (not `.as_ref()`) due to palette feature conflicts
- For non-blocking async in TUI: spawn with `tokio::spawn`, check `is_finished()` each loop iteration
- Extract rendering/large functions to helpers to stay under clippy's 100-line function limit
- Use `dirs::cache_dir()` for platform-specific cache paths (macOS: ~/Library/Caches, Linux: ~/.cache)
